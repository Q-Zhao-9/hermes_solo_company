Package creation is confirmation-gated and only includes published public docs that explicitly target the selected framework. Portable bundles include published public docs for migration/import review.

Deployment handoff

Preview and package a deployment handoff ZIP for reviewed docs artifacts. This never publishes externally by itself.

Deployment handoff is review-first and confirmation-gated with confirmDeploymentPackage:true. It does not push to Sitelet, WordPress, hosting, DNS, or production automatically.

Deployment audit operations

Review, filter, summarize, and export the local audit log of confirmed deployment handoff ZIP packages. History is owner-protected and never stores auth tokens.

The audit log records package metadata such as target, environment, locale, path, size, approver, manifest summary, and timestamp.

Deployment package operations

Inspect a package manifest, re-download a local ZIP, compare two handoff packages, and update manual checklist status. These operations are owner-protected and local-only.

Package operations never deploy externally; they only inspect/download existing local ZIPs and track manual deployment checklist progress.

Deployment approval workflow

Review package approval status, generate release notes, and lock approved/released handoff packages from further checklist mutation.

Approval workflow is local metadata only. It does not publish, upload, deploy, or call external services.

Release dashboard

Review deployment packages by approval state, readiness scoring, release queue, and generate an operator handoff report.

Release dashboard and handoff report are local review artifacts only; they do not deploy externally.

Release archive

Create immutable local release attestations with SHA-256 package, manifest, and handoff report hashes. Download archived operator reports for human deployment handoff.

Release archive is local attestation metadata only. It does not deploy, publish, upload, or call external services.

Release restore / rollback

Verify archived release integrity, generate rollback plans, and prepare local restore packages for operator review.

Restore packages are local-only planning artifacts. They do not deploy, publish, upload, or call external services.

Deployment connector dry-run

Run local-only connector preflight checks for Sitelet, WordPress, and static hosting handoff readiness. Phase 21 adds Connector profiles and dry-run history without calling external APIs.

Connector runbooks and comparison

Phase 22 generates local-only Connector runbooks and compares dry-run results for operator review.

Connector profiles store secret placeholders only. Connector dry-runs and Connector runbooks are local-only, redact secrets, and do not deploy, publish, upload, rollback, or call external services.

Final operator release playbooks

Phase 23 creates local-only human operator playbooks for Sitelet, WordPress, static hosting, and framework deployment handoff.

Packaging and onboarding

Phase 24 generates local-only install, environment, backup/restore, Sitelet, WordPress, and admin workflow onboarding guides.

Final QA and v1 release

Phase 25 generates the final local-only QA summary, release freeze checklist, and confirmation-gated v1 release package.

Import docs

Paste a JSON array of files from Markdown folders, Docusaurus, MkDocs, VitePress, Hugo, or an Easiio portable bundle. Preview detects slug conflicts before execution.

Import execution is owner-protected and requires browser confirmation plus confirmImport:true.
